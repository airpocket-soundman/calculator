# Calculator

GitHub Pages で公開する、PWA 対応の計算機アプリです。

この README は、実装前提のメモではなく、このリポジトリで作るアプリの目的、仕様、公開方針、開発手順を整理したものです。

## 目的

- ブラウザで動作するシンプルな計算機を提供する
- GitHub Pages で静的公開できる構成にする
- PWA 化して、スマートフォンや PC でホーム画面追加やオフライン利用を可能にする

## 公開方針

- 公開先は GitHub Pages
- 静的ファイルのみで動作する構成にする
- `main` ブランチへの反映後に GitHub Pages へデプロイできる状態を目指す

## アプリ仕様

### 基本機能

- 数字入力
- 四則演算: `+`, `-`, `*`, `/`
- 括弧入力: `(`, `)`
- 小数点入力
- 符号切り替え
- パーセント計算
- クリア
- 直前入力の削除
- イコールによる計算結果表示

### 関数電卓機能

- べき乗: `x^y`
- 平方根
- 任意乗根
- 指数関数
- 対数: `log`, `ln`
- 三角関数: `sin`, `cos`, `tan`
- 逆三角関数
- 角度単位の切り替え: `DEG` / `RAD`
- 定数: `pi`, `e`
- 括弧を使った式入力
- メモリ機能: `MC`, `MR`, `M+`, `M-`
- パーセント計算
- 階乗
- 直前計算の再利用

### 計算仕様

- 一般的な関数電卓に近い操作感を持つこと
- 演算子の優先順位を考慮して計算すること
- 括弧を含む式を正しく評価できること
- ゼロ除算や不正入力時はエラー表示を行うこと
- 極端に長い小数は表示桁数を制御すること
- 必要に応じて指数表記を使うこと

### 表示仕様

- 現在入力中の値を見やすく表示する
- 計算式または直前の演算内容を補助表示できる構成にする
- 桁数が多い場合でもレイアウトが崩れないこと

### 操作仕様

- マウス操作に対応
- スマートフォンのタップ操作に対応
- キーボード入力にも対応する
- 左右カーソルキーで入力位置を移動できること
- `BS` でカーソル直前の文字を削除できること
- `Del` でカーソル位置の文字を削除できること
- 途中位置にカーソルを移動して式を修正できること
- 誤操作しにくいボタンサイズと余白を確保する

## PWA 要件

- `manifest.json` を用意する
- `service worker` を設定する
- ホーム画面に追加できること
- オフライン時でも少なくともアプリ本体が起動できること
- アイコンを複数サイズで用意する

## GitHub Pages 対応要件

- 相対パスまたは Pages 配下で壊れないパス設計にする
- 静的ホスティング前提で動作すること
- 追加のサーバーサイド処理を必要としないこと

## 想定ディレクトリ構成

実装時は、最低限以下のような構成を想定します。

```text
.
├─ index.html
├─ styles.css
├─ app.js
├─ manifest.json
├─ sw.js
└─ assets/
   └─ icons/
```

## 開発方針

- 依存を増やしすぎず、まずは素の HTML / CSS / JavaScript で成立する構成を優先する
- 先に関数電卓としての計算ロジックを安定させ、その後 UI と PWA 対応を追加する
- GitHub Pages でそのまま配信できるよう、ビルド不要または軽量な構成を優先する

## 開発タスク

1. 関数電卓向けの UI を作成する
2. 式入力と演算子優先順位を含む計算ロジックを実装する
3. 関数群とメモリ機能を実装する
4. キーボード操作と式のカーソル編集を実装する
5. `manifest.json` を追加する
6. `service worker` を追加してオフライン対応する
7. GitHub Pages で公開する
8. モバイル表示とインストール導線を確認する

## ローカルテスト方法

Windows の PowerShell で、このリポジトリのディレクトリを開いた状態で確認します。

このリポジトリには Node.js ベースの簡易ローカルサーバーを同梱しています。Python は不要です。

### 1. ローカルサーバーを起動する

以下を実行します。

```powershell
npm run serve
```

### 2. ブラウザで開く

以下の URL を開きます。

```text
http://localhost:8000
```

### 3. 確認する内容

- 計算機の画面が表示されること
- 数字、演算子、括弧、関数キーが操作できること
- 左右カーソルキーで入力位置を移動できること
- `BS` と `DEL` で式を編集できること
- `=` または Enter で計算できること
- `DEG` / `RAD` 切り替えが反映されること
- `MC`, `MR`, `M+`, `M-` が動作すること

### 4. PWA の確認

- ブラウザで `manifest` が読み込まれていること
- `service worker` が登録されること
- インストール可能なブラウザではインストール導線が表示されること
- 一度読み込んだ後、オフラインでもアプリ本体が開けること

サーバー起動後に、以下のような表示が出れば成功です。

```text
Server running at http://localhost:8000
```

注意:

- `index.html` を直接ダブルクリックして開く方法では、PWA 機能の確認が不完全になります
- `manifest.json` や `service worker` を確認するため、必ず `http://localhost:8000` のようなローカルサーバー経由で開いてください

## GitHub Pages 設定方法

### 前提

- GitHub にこのリポジトリを push 済みであること
- 公開したい内容が `main` ブランチにあること

### 設定手順

1. GitHub で対象リポジトリを開く
2. `Settings` を開く
3. 左メニューの `Pages` を開く
4. `Build and deployment` の `Source` で `Deploy from a branch` を選ぶ
5. Branch に `main` を選ぶ
6. Folder に `/ (root)` を選ぶ
7. `Save` を押す

### 公開後の確認

- 数分待ってから、GitHub Pages の公開 URL を開く
- 計算機の画面が正しく表示されることを確認する
- 関数電卓機能が動作することを確認する
- スマートフォン表示でもレイアウトが崩れていないことを確認する
- PWA としてインストールできることを確認する

### 注意点

- リポジトリ名付きの GitHub Pages URL でも動くよう、相対パスで構成しています
- `main` ブランチ更新後、Pages 反映まで少し時間がかかることがあります
- `service worker` のキャッシュが残る場合があるため、更新確認時はハードリロードやキャッシュ削除も必要です
